How to Build Your First React App in 2025


React has become one of the most popular JavaScript libraries for building user interfaces, and its community continues to grow exponentially. Whether you’re a beginner exploring the world of front-end development or a seasoned developer looking to dive into the React ecosystem, building your first React app is a great place to start.

In this guide, we’ll walk you through the steps required to build your very first React app in 2025. We’ll cover everything from setting up your development environment to creating components and managing state. By the end, you’ll have a solid understanding of how React works and how to start building sophisticated apps.

Prerequisites

Before we dive into the specifics of React, there are a few prerequisites you’ll need:

  • Basic knowledge of JavaScript: React is built on JavaScript, so it’s essential to understand JavaScript fundamentals such as variables, functions, loops, and objects.
  • Basic HTML and CSS: Understanding HTML and CSS is important because React uses JSX (a syntax extension to JavaScript) that closely resembles HTML. Styling your components is also important for making your app visually appealing.
  • Node.js: React relies on Node.js for its build tools and package management. You’ll need to have Node.js installed on your computer to proceed.

Step 1: Setting Up Your Development Environment

To get started with React, you’ll need to set up your development environment. Fortunately, React’s official tooling makes this process easy. The simplest way to set up a new React project is by using Create React App, a command-line tool developed by Facebook.

Installing Node.js and npm

If you don’t already have Node.js installed, head over to the official Node.js website and download the latest version. Installing Node.js will also install npm (Node Package Manager), which is used to manage project dependencies.

To verify that Node.js and npm are correctly installed, open your terminal and run:

node -v
npm -v

Both commands should return version numbers for Node.js and npm.

Installing Create React App

Once Node.js and npm are installed, you can install Create React App globally by running the following command:

npm install -g create-react-app

This tool provides an easy way to set up a new React project without having to configure webpack, Babel, and other build tools manually.

Creating Your First React App

To create a new React project, navigate to the directory where you want to store your project and run:

npx create-react-app my-first-react-app

Replace my-first-react-app with the name of your app. The npx command ensures that you are always using the latest version of create-react-app.

Once the setup is complete, navigate into your new project directory:

cd my-first-react-app

You can now start the development server by running:

npm start

This will open a local server at http://localhost:3000, and you’ll see your first React app running in the browser. Congratulations on setting up your environment!

Step 2: Understanding the Project Structure

Once you’ve created your React app, let’s take a quick look at the folder structure that was generated by Create React App:

my-first-react-app/
├── node_modules/              # Project dependencies
├── public/
│   ├── index.html             # HTML template
│   └── ...
├── src/
│   ├── App.js                 # Main App component
│   ├── index.js               # Entry point for the React app
│   └── ...
├── package.json               # Project metadata and dependencies
└── ...

Here’s a breakdown of the most important files:

  • public/index.html: The single HTML file that serves as the template for your app.
  • src/index.js: This is the entry point for your React app. It renders the root component (<App />) into the div#root element of index.html.
  • src/App.js: This file contains the main component of your app. By default, it displays some introductory text, but we’ll customize it later.

Step 3: Exploring React Components

At the core of React are components. Components are JavaScript functions or classes that return JSX (a syntax that resembles HTML) and define how the UI should appear. Let’s take a closer look at how React components work.

Creating Your First Component

Open the src/App.js file and replace the default content with the following:

import React from 'react';

function App() {
  return (
    <div>
      <h1>Welcome to My First React App!</h1>
      <p>This is my first component in React.</p>
    </div>
  );
}

export default App;

In this example:

  • App is a functional component, which is the most common type of component in modern React development.
  • The JSX inside the return statement is how we define the structure of the UI. It’s similar to HTML but has a few differences, such as using className instead of class for CSS classes.

Once you save the file, the changes will automatically be reflected in the browser.

Component Hierarchy

React apps are built by combining components together in a hierarchy. For example, if you wanted to create a component for a header, you could create a new file called Header.js inside the src folder.

Here’s what the Header.js component might look like:

import React from 'react';

function Header() {
  return <h1>My First React App</h1>;
}

export default Header;

Now, you can import and use this new Header component inside your App.js component:

import React from 'react';
import Header from './Header';

function App() {
  return (
    <div>
      <Header />
      <p>Welcome to my first React app!</p>
    </div>
  );
}

export default App;

This is the basic structure of how React apps are built by composing small, reusable components.

Step 4: Understanding JSX

JSX (JavaScript XML) is a syntax extension to JavaScript that looks similar to HTML. It allows you to write UI elements in JavaScript and combines the best parts of JavaScript and HTML.

Key Points About JSX:

  1. JSX is not HTML: JSX looks like HTML, but it’s not. Under the hood, it gets transformed into JavaScript code that React understands.

  2. JSX Expressions: You can embed JavaScript expressions inside JSX by wrapping them in curly braces ({}). For example:

const name = 'John';
const greeting = <p>Hello, {name}!</p>;

  1. Class vs. className: In JSX, class is replaced with className to avoid conflicts with the class keyword in JavaScript.

  2. Self-closing tags: Just like in XML, self-closing tags in JSX (e.g., <img />, <input />) need to be explicitly closed.

Step 5: Working with State

One of the key features of React is its ability to manage and update the state of components. State allows components to store and change information over time. Let’s explore how to manage state in your app.

Adding State to Your App

React provides a useState hook for managing state in functional components. Here’s how you can modify your App.js to use state:

import React, { useState } from 'react';

function App() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>Count: {count}</h1>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default App;

In this example:

  • useState(0) initializes the count state variable with a value of 0.
  • The setCount function updates the state whenever the button is clicked.

Step 6: Handling Events

React provides a declarative way to handle events such as button clicks, form submissions, and keyboard interactions. In the previous example, you saw how to handle a button click using the onClick event.

Here’s an example of handling a form submission:

import React, { useState } from 'react';

function App() {
  const [name, setName] = useState('');

  const handleSubmit = (e) => {
    e.preventDefault();
    alert(`Hello, ${name}!`);
  };

  return (
    <div>
      <form onSubmit={handleSubmit}>
        <input 
          type="text" 
          value={name} 
          onChange={(e) => setName(e.target.value)} 
          placeholder="Enter your name"
        />
        <button type="submit">Submit</button>
      </form>
    </div>
  );
}

export default App;

In this example:

  • The onChange event updates the state every time the user types in the input field.
  • The onSubmit event triggers when the form is submitted.

Step 7: Styling Your React App

React allows you to style components in a variety of ways. You can use traditional CSS, CSS-in-JS, or even CSS modules for scoped styling. For now, let’s stick to regular CSS.

In your src folder, create a styles.css file and add the following:

h1 {
  color: #333;
}

button {
  background-color: blue;
  color: white;
  padding: 10px;
  border: none;
  cursor: pointer;
}

Now, import the CSS file into your App.js:

import './styles.css';

Once the styles are imported, they will be applied to the components.

Step 8: Final Thoughts

Congratulations! You’ve now built your first React app in 2025. You’ve learned how to set up your development environment, create React components, manage state, handle events, and style your app. With these fundamental skills, you’re ready to take on more advanced React features, such as routing, state management with Redux or React Context, and working with APIs.

Remember, React is a powerful library, and the best way to become proficient is by building real-world projects. Keep experimenting, and you’ll continue to improve your skills. Happy coding!
