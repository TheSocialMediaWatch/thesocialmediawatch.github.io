How to Build a Simple To-Do App with React and Firebase


In today’s fast-paced world, to-do apps have become essential tools for managing daily tasks, organizing workflows, and boosting productivity. If you’re looking to build a simple yet functional to-do application, you might want to consider using React and Firebase. React, a popular JavaScript library for building user interfaces, offers an intuitive way to create dynamic and responsive web apps. Firebase, on the other hand, provides a real-time database and powerful backend services that make building and deploying applications much easier.

In this tutorial, we’ll walk through the process of creating a simple to-do app using React and Firebase. By the end of this guide, you’ll have a fully functional app where users can add, edit, delete, and mark tasks as complete. Additionally, Firebase will handle the data storage, enabling your app to persist tasks across sessions.

Prerequisites

Before we dive into building the app, here’s a list of things you’ll need:

  1. Basic knowledge of JavaScript and React: You should be familiar with the fundamentals of JavaScript, React components, and hooks.
  2. Firebase account: You need to create a Firebase project in order to access its services. Don’t worry if you don’t have one yet, as we’ll guide you through the setup process.
  3. Node.js and npm: Make sure you have Node.js and npm installed on your machine, as these tools are necessary to create and manage the React project.

If you don’t have these tools installed yet, visit the following sites to get started:

  • Node.js
  • Firebase

Step 1: Setting Up Your React Project

The first step in creating your to-do app is setting up a React project. If you haven’t already created one, follow these steps:

  1. Install Create React App (if you don’t have it installed yet). Open your terminal and run:

    npx create-react-app todo-app

  2. Once the installation is complete, navigate into your project folder:

    cd todo-app

  3. Start the development server by running:

    npm start

This should open your default browser with the React app running at http://localhost:3000. You’ll see the default Create React App welcome page.

Step 2: Setting Up Firebase

Firebase offers several tools, but for our to-do app, we’ll mainly focus on Firebase’s Firestore database. Here’s how you can set it up:

  1. Go to the Firebase Console and log in with your Google account.
  2. Click Add Project and follow the on-screen instructions to create a new Firebase project.
  3. Once your project is created, click on Firestore Database from the left-hand menu.
  4. Click on Create Database, and select Start in test mode for development purposes.
  5. After creating the database, click on the Settings gear in the Firebase console and go to Project Settings.
  6. Under the General tab, find your Firebase SDK setup and configuration. You’ll need this to integrate Firebase with your React app.

Copy the Firebase config object. It will look something like this:

const firebaseConfig = {
  apiKey: "your-api-key",
  authDomain: "your-project-id.firebaseapp.com",
  projectId: "your-project-id",
  storageBucket: "your-project-id.appspot.com",
  messagingSenderId: "your-sender-id",
  appId: "your-app-id",
  measurementId: "your-measurement-id"
};

Step 3: Install Firebase SDK

Now that you’ve set up Firebase, you need to install the Firebase SDK in your React app. In your terminal, navigate to your React project folder and run the following command:

npm install firebase

This will add the Firebase SDK to your project. Now, let’s integrate Firebase with the app.

Step 4: Setting Up Firebase in Your React App

Create a new file called firebase.js inside the src folder of your React app to initialize Firebase. Paste the Firebase config object you copied earlier into this file and initialize Firebase like so:

import firebase from 'firebase/app';
import 'firebase/firestore';

// Initialize Firebase with the config object
const firebaseConfig = {
  apiKey: "your-api-key",
  authDomain: "your-project-id.firebaseapp.com",
  projectId: "your-project-id",
  storageBucket: "your-project-id.appspot.com",
  messagingSenderId: "your-sender-id",
  appId: "your-app-id",
  measurementId: "your-measurement-id"
};

const firebaseApp = firebase.initializeApp(firebaseConfig);
const db = firebaseApp.firestore();

export { db };

Now your app is ready to communicate with Firebase.

Step 5: Building the To-Do App Components

App Structure

We’ll organize our app with the following components:

  • App.js: The main component that will display the list of tasks.
  • TodoItem.js: A component to represent each individual to-do item.
  • AddTodo.js: A component for adding new tasks.
  • TodoList.js: A component to list all the to-dos.

Let’s start by building the basic structure.

App Component (App.js)

In App.js, we’ll import the necessary components and display the to-do list and add task form.

import React, { useState, useEffect } from 'react';
import './App.css';
import { db } from './firebase';
import TodoItem from './TodoItem';
import AddTodo from './AddTodo';

const App = () => {
  const [todos, setTodos] = useState([]);

  useEffect(() => {
    // Fetch todos from Firestore when the app loads
    db.collection('todos')
      .orderBy('timestamp')
      .onSnapshot(snapshot => {
        setTodos(snapshot.docs.map(doc => ({ id: doc.id, data: doc.data() })));
      });
  }, []);

  return (
    <div className="App">
      <h1>Simple To-Do App</h1>
      <AddTodo />
      <ul>
        {todos.map(todo => (
          <TodoItem key={todo.id} todo={todo} />
        ))}
      </ul>
    </div>
  );
};

export default App;

TodoItem Component (TodoItem.js)

This component will display an individual to-do item. Each task will have a checkbox to mark it as completed, and a delete button to remove it.

import React from 'react';
import { db } from './firebase';

const TodoItem = ({ todo }) => {
  const handleDelete = () => {
    db.collection('todos').doc(todo.id).delete();
  };

  const handleToggleComplete = () => {
    db.collection('todos').doc(todo.id).update({
      completed: !todo.data.completed
    });
  };

  return (
    <li>
      <input
        type="checkbox"
        checked={todo.data.completed}
        onChange={handleToggleComplete}
      />
      {todo.data.task}
      <button onClick={handleDelete}>Delete</button>
    </li>
  );
};

export default TodoItem;

AddTodo Component (AddTodo.js)

This component will allow users to add new to-do tasks. When the user submits the form, a new task will be added to the Firestore database.

import React, { useState } from 'react';
import { db } from './firebase';

const AddTodo = () => {
  const [task, setTask] = useState('');

  const handleSubmit = (e) => {
    e.preventDefault();

    if (task) {
      db.collection('todos').add({
        task: task,
        completed: false,
        timestamp: firebase.firestore.FieldValue.serverTimestamp(),
      });

      setTask('');
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        value={task}
        onChange={(e) => setTask(e.target.value)}
        placeholder="Add a new task"
      />
      <button type="submit">Add Task</button>
    </form>
  );
};

export default AddTodo;

Step 7: Running the App

Now, if you run the app using npm start, you should be able to:

  • Add new tasks.
  • View tasks in a list.
  • Mark tasks as complete.
  • Delete tasks.

Conclusion

Congratulations! You’ve built a simple yet functional to-do app using React and Firebase. In this tutorial, you learned how to:

  • Set up a React project and integrate it with Firebase.
  • Create a Firestore database to store tasks.
  • Build React components to display tasks, add new ones, and handle task updates.
  • Implement real-time data syncing with Firestore.

This is just the beginning. You can extend this app by adding features like authentication, user-specific tasks, or even deploying it to Firebase Hosting. The possibilities are endless!
