How to Create a REST API with Spring Boot


Creating a REST API is an essential skill for modern application development. With the rise of microservices and distributed architectures, RESTful services have become the backbone of many web applications. Spring Boot, a powerful framework of the Spring ecosystem, simplifies the process of building production-ready REST APIs with minimal configuration and great flexibility. In this comprehensive guide, we’ll dive deep into how to create a REST API using Spring Boot from scratch, covering everything from project setup to handling different HTTP methods, validations, error handling, and best practices for building scalable and maintainable APIs.

Understanding REST and Spring Boot

Before jumping into code, it helps to briefly understand what makes a REST API. REST (Representational State Transfer) is an architectural style that uses standard HTTP methods like GET, POST, PUT, DELETE, etc., to perform CRUD (Create, Read, Update, Delete) operations on resources identified by URLs.

Spring Boot is an opinionated framework that simplifies Spring development by automating configurations and providing ready-made components to rapidly build applications. Its integration with Spring MVC and features such as embedded servers, starter dependencies, and automatic configuration make it ideal for creating RESTful APIs quickly and efficiently.

Setting Up Your Spring Boot Project

To create a REST API with Spring Boot, start by setting up a new project. You have multiple options including:

  • Using Spring Initializr (web-based)
  • Creating a Maven or Gradle project manually
  • Using an IDE such as IntelliJ IDEA or Eclipse with Spring plugins

Using Spring Initializr

  1. Go to Spring Initializr.
  2. Choose your Project: Maven or Gradle.
  3. Select Language: Java.
  4. Choose Spring Boot version (preferably the latest stable release).
  5. Add Dependencies:
    • Spring Web (includes Spring MVC for REST API development)
    • Spring Data JPA (for database interaction if needed)
    • H2 Database (in-memory database for testing)
    • Lombok (optional, for reducing boilerplate code)
  6. Generate and download the project archive.
  7. Unzip into your workspace and open in your favorite IDE.

Defining the Domain Model

Let’s consider building a simple API for managing Books. Each book will have an ID, title, author, and a publication year.

package com.example.demo.model;

import jakarta.persistence.Entity;
import jakarta.persistence.GeneratedValue;
import jakarta.persistence.GenerationType;
import jakarta.persistence.Id;
import jakarta.validation.constraints.NotBlank;
import jakarta.validation.constraints.NotNull;

@Entity
public class Book {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @NotBlank(message = "Title is required")
    private String title;

    @NotBlank(message = "Author is required")
    private String author;

    @NotNull(message = "Publication year is required")
    private Integer year;

    // Getters and setters omitted for brevity
}

Explanation

  • The class is annotated with @Entity to define a JPA entity mapped to a database table.
  • @Id and @GeneratedValue handle primary key generation.
  • Validation annotations such as @NotBlank and @NotNull enforce required fields when receiving input in the API requests.

Creating the Repository Layer

Interacting with the database is simplified with Spring Data JPA repositories. You only need to declare interfaces extending Spring’s JpaRepository to get CRUD functionality.

package com.example.demo.repository;

import com.example.demo.model.Book;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.stereotype.Repository;

@Repository
public interface BookRepository extends JpaRepository<Book, Long> {
    // Custom query methods can be defined here if needed
}

This interface inherits methods like findAll(), findById(), save(), and deleteById(), allowing us to focus on higher-level business logic.

Building the Service Layer

Although small applications may skip this layer, it’s a good practice to introduce a service abstraction between controllers and repositories. It encapsulates business logic and promotes cleaner, reusable code.

package com.example.demo.service;

import com.example.demo.model.Book;
import com.example.demo.repository.BookRepository;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.Optional;

@Service
public class BookService {

    @Autowired
    private BookRepository bookRepository;

    public List<Book> getAllBooks() {
        return bookRepository.findAll();
    }

    public Optional<Book> getBookById(Long id) {
        return bookRepository.findById(id);
    }

    public Book saveBook(Book book) {
        return bookRepository.save(book);
    }

    public void deleteBook(Long id) {
        bookRepository.deleteById(id);
    }
}

This service defines straightforward methods to support CRUD operations on the Book entity.

Creating the REST Controller

Controllers process incoming HTTP requests, call services, and return responses. Spring Boot’s @RestController combines @Controller and @ResponseBody making it simple to build REST endpoints.

package com.example.demo.controller;

import com.example.demo.model.Book;
import com.example.demo.service.BookService;
import jakarta.validation.Valid;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/api/books")
public class BookController {

    @Autowired
    private BookService bookService;

    // Get all books
    @GetMapping
    public List<Book> getBooks() {
        return bookService.getAllBooks();
    }

    // Get book by ID
    @GetMapping("/{id}")
    public ResponseEntity<Book> getBook(@PathVariable Long id) {
        return bookService.getBookById(id)
                .map(book -> ResponseEntity.ok(book))
                .orElse(ResponseEntity.notFound().build());
    }

    // Create a new book
    @PostMapping
    @ResponseStatus(HttpStatus.CREATED)
    public Book createBook(@Valid @RequestBody Book book) {
        return bookService.saveBook(book);
    }

    // Update a book
    @PutMapping("/{id}")
    public ResponseEntity<Book> updateBook(@PathVariable Long id, @Valid @RequestBody Book bookDetails) {
        return bookService.getBookById(id)
                .map(book -> {
                    book.setTitle(bookDetails.getTitle());
                    book.setAuthor(bookDetails.getAuthor());
                    book.setYear(bookDetails.getYear());
                    Book updatedBook = bookService.saveBook(book);
                    return ResponseEntity.ok(updatedBook);
                })
                .orElse(ResponseEntity.notFound().build());
    }

    // Delete a book
    @DeleteMapping("/{id}")
    public ResponseEntity<Void> deleteBook(@PathVariable Long id) {
        return bookService.getBookById(id)
                .map(book -> {
                    bookService.deleteBook(id);
                    return new ResponseEntity<Void>(HttpStatus.NO_CONTENT);
                })
                .orElse(ResponseEntity.notFound().build());
    }
}

Controller Breakdown

  • @GetMapping returns a list or a specific book by ID.
  • @PostMapping creates new records, validating input with @Valid.
  • @PutMapping updates an existing book by retrieving it first, then saving changes.
  • @DeleteMapping removes the book if it exists.
  • HTTP status codes like 200 OK, 201 Created, 204 No Content, and 404 Not Found are appropriately used.
  • ResponseEntity allows fine-tuning of responses including headers and status codes.

Handling Input Validation and Errors

Input validation is vital to preventing invalid data from entering the system. Spring Boot supports JSR-380 Bean Validation with the help of annotations like @NotBlank, combined with @Valid in controller parameters.

If validation fails, Spring automatically returns a 400 Bad Request with details on the violated constraints. You can customize this behavior by defining an exception handler:

package com.example.demo.exception;

import org.springframework.http.HttpHeaders;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.validation.FieldError;
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.ControllerAdvice;
import org.springframework.web.context.request.WebRequest;
import org.springframework.web.servlet.mvc.method.annotation.ResponseEntityExceptionHandler;

import java.util.HashMap;
import java.util.Map;

@ControllerAdvice
public class GlobalExceptionHandler extends ResponseEntityExceptionHandler {

    @Override
    protected ResponseEntity<Object> handleMethodArgumentNotValid(
        MethodArgumentNotValidException ex,
        HttpHeaders headers,
        HttpStatus status,
        WebRequest request) {

        Map<String, String> errors = new HashMap<>();
        ex.getBindingResult().getAllErrors().forEach((error) -> {
            String fieldName = ((FieldError) error).getField();
            String errorMessage = error.getDefaultMessage();
            errors.put(fieldName, errorMessage);
        });
        return new ResponseEntity<>(errors, HttpStatus.BAD_REQUEST);
    }
}

This returns a structured JSON response highlighting each invalid field and the message explaining why it failed validation.

Further Enhancements

Once you have a working API, consider these improvements:

1. Pagination and Sorting

For large datasets, adding pagination and sorting improves performance and usability.

Spring Data JPA supports pageable parameters easily in repository methods:

Page<Book> findAll(Pageable pageable);

Modify your controller to accept parameters like page, size, and sort.

2. API Documentation

Generating API documentation automatically with Swagger (OpenAPI) helps front-end developers and API consumers.

Add the springdoc-openapi-ui dependency and annotate your controllers. This generates an interactive UI at /swagger-ui.html.

3. Security

Secure your API to protect sensitive data:

  • Add Spring Security for authentication and authorization.
  • Implement token-based authentication such as JWT.
  • Use HTTPS to encrypt traffic.

4. CORS

Configure Cross-Origin Resource Sharing (CORS) if your API will be accessed from web browsers hosted on different origins.

@CrossOrigin(origins = "http://example.com")
@RestController
@RequestMapping("/api/books")
public class BookController {
    //...
}

5. Exception Handling Enhancements

Create custom exceptions for resource not found, invalid requests, and implement centralized handling for cleaner API responses.

Deploying the Spring Boot REST API

Spring Boot applications package as self-contained JARs with embedded servers like Tomcat. Deployment options include:

  • Running locally with java -jar yourapp.jar
  • Deploying on cloud platforms such as AWS, Azure, or Google Cloud.
  • Containerizing with Docker for easier orchestration and deployment.
  • Using CI/CD pipelines to automate builds and deployment.

Conclusion

Building a REST API with Spring Boot provides a solid foundation for scalable, maintainable, and production-ready web services. Leveraging Spring Boot’s auto-configuration, integration with Spring MVC, and Spring Data JPA reduces boilerplate and accelerates development.

From setting up the project, designing the domain, creating repositories and services, to developing REST controllers and handling validation and error scenarios, each step builds toward a robust API. As your application grows, adding features like pagination, security, documentation, and better error management will enhance the API’s maturity.

This guide offers a practical, end-to-end overview to get your RESTful service up and running with Spring Boot. Whether you’re building a simple CRUD backend or a complex microservice, Spring Boot makes it not just easier, but enjoyable.

Explore the endless possibilities of your Spring Boot REST API and keep iterating to deliver powerful backend solutions that clients and users will love.
