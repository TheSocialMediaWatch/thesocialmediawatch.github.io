How to Create a Web Application with Flask and React


Creating a web application using Flask and React is a powerful combination that leverages the simplicity of Python for the backend and the dynamic, user-friendly interface of React for the frontend. Flask is a lightweight web framework that provides the tools necessary to build robust web applications, while React is a popular JavaScript library for building interactive user interfaces.

In this guide, we will walk you through the process of setting up a Flask backend and a React frontend, demonstrating how to create a web application that can handle requests and serve dynamic content to the user. Whether you are a beginner or an experienced developer, this tutorial will help you understand the workflow between Flask and React, and how to integrate them into a seamless web application.

Step 1: Setting Up the Flask Backend

Before diving into the React part of the application, let’s start by setting up the Flask backend. Flask will be responsible for handling HTTP requests, interacting with databases, and serving data to the frontend. For simplicity, we’ll create a basic REST API.

1.1 Install Flask

First, make sure you have Python installed on your system. You can check this by running the following command in your terminal:

python --version

If Python is installed, you can proceed to install Flask. It’s recommended to create a virtual environment to manage dependencies. Run the following commands:

python -m venv venv
source venv/bin/activate  # For Linux/Mac
venv\Scripts\activate  # For Windows

Next, install Flask:

pip install Flask

1.2 Create a Basic Flask Application

Create a new directory for your Flask project. Inside the directory, create a file named app.py. This file will serve as the entry point for your Flask application. Here’s a simple Flask app that returns a JSON response:

from flask import Flask, jsonify

app = Flask(__name__)

@app.route('/api', methods=['GET'])
def get_data():
    data = {
        "message": "Hello from Flask!"
    }
    return jsonify(data)

if __name__ == '__main__':
    app.run(debug=True)

This code creates a basic Flask application with one route (/api) that returns a JSON response with a message. You can run the app by executing the following command:

python app.py

The Flask app will be available at http://127.0.0.1:5000/api.

1.3 Setting Up CORS

When the frontend React application tries to communicate with the Flask backend, it may encounter Cross-Origin Resource Sharing (CORS) issues. CORS is a security feature implemented by browsers to prevent websites from making requests to a different domain. To resolve this, you need to enable CORS in your Flask app.

Install the flask-cors package:

pip install flask-cors

Now, modify your app.py to include CORS:

from flask import Flask, jsonify
from flask_cors import CORS

app = Flask(__name__)
CORS(app)  # Enable CORS for all routes

@app.route('/api', methods=['GET'])
def get_data():
    data = {
        "message": "Hello from Flask!"
    }
    return jsonify(data)

if __name__ == '__main__':
    app.run(debug=True)

With this change, your Flask app will be able to handle requests from different origins, such as your React frontend.

Step 2: Setting Up the React Frontend

Now that the backend is ready, let’s move on to creating the React frontend. React is a powerful JavaScript library for building interactive user interfaces. It allows developers to create reusable components and manage application state efficiently.

2.1 Install Node.js and Create a React App

Before you start working with React, make sure you have Node.js installed. You can check if Node.js is installed by running:

node -v

If Node.js is not installed, visit Node.js website and download the installer for your operating system.

Once Node.js is installed, you can create a new React app using create-react-app, a tool that sets up a React project with a good default configuration. Run the following command:

npx create-react-app frontend

This will create a new React app in a directory called frontend. Once the installation is complete, navigate into the frontend directory:

cd frontend

To run the React development server, execute:

npm start

This will start a development server and open your React app in the browser at http://localhost:3000.

2.2 Create a Simple React Component

Let’s modify the default React app to fetch data from the Flask backend. Open the src/App.js file and update it to the following code:

import React, { useEffect, useState } from 'react';

function App() {
  const [data, setData] = useState(null);

  useEffect(() => {
    fetch('http://127.0.0.1:5000/api')
      .then(response => response.json())
      .then(data => setData(data));
  }, []);

  return (
    <div className="App">
      <h1>React and Flask Integration</h1>
      {data ? (
        <p>{data.message}</p>
      ) : (
        <p>Loading data from Flask...</p>
      )}
    </div>
  );
}

export default App;

This code does the following:

  • It uses the useEffect hook to make an HTTP GET request to the Flask backend when the component mounts.
  • It stores the response data in the data state using the useState hook.
  • It displays the message from the Flask backend in the browser.

2.3 Running React and Flask Together

Now that both the Flask backend and the React frontend are set up, you need to run both servers simultaneously. Here’s how to do it:

  1. Run the Flask server by executing python app.py in the backend directory.
  2. Run the React development server by executing npm start in the frontend directory.

Your Flask backend will be running on http://127.0.0.1:5000 and the React frontend will be running on http://localhost:3000. When you open the React app in the browser, it will fetch data from the Flask backend and display it.

Step 3: Building for Production

Once you’ve built your web application and everything is working as expected, it’s time to prepare for production. This involves optimizing both the Flask backend and the React frontend for deployment.

3.1 Build the React App

To create a production build of your React app, run the following command:

npm run build

This will generate a build directory containing optimized static files (HTML, CSS, JavaScript) that can be served by the Flask backend.

3.2 Serve React Build with Flask

To serve the React build with Flask, you can modify your app.py to point to the static files generated by React. First, move the build directory to your Flask project directory, and then update your Flask app as follows:

import os
from flask import Flask, send_from_directory
from flask_cors import CORS

app = Flask(__name__, static_folder='build', static_url_path='')

CORS(app)

@app.route('/api', methods=['GET'])
def get_data():
    data = {
        "message": "Hello from Flask!"
    }
    return jsonify(data)

@app.route('/', defaults={'path': ''})
@app.route('/<path:path>')
def serve_react_app(path):
    return send_from_directory(app.static_folder, 'index.html')

if __name__ == '__main__':
    app.run(debug=False)

In this setup:

  • The Flask app serves the static files from the build directory.
  • Any request to a route other than /api will be served the index.html file from the React build, ensuring the React app can handle the routing on the client side.

3.3 Deploying to Production

For production deployment, you can choose from a variety of hosting platforms, such as AWS, Heroku, or DigitalOcean. Each platform provides documentation on how to deploy Flask and React applications.

When deploying, you’ll want to ensure that the Flask app is configured to run behind a production-grade web server like Gunicorn, and that the React build files are served correctly.

Conclusion

By combining Flask and React, you’ve built a powerful full-stack web application. Flask handles the backend logic, while React creates an interactive user interface. You’ve learned how to set up both the Flask backend and React frontend, integrate them, and deploy them for production.

With Flask and React, you can create scalable and dynamic web applications with minimal complexity. Whether you’re building a simple web app or a more complex system, this combination gives you the flexibility and control you need to develop modern, high-performance applications.

Happy coding!
