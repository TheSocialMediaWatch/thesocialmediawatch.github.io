How to Enable Dark Mode Based on User Preferences


Dark mode has become a popular feature in web design and app development due to its ability to reduce eye strain, save battery life, and create a sleek, modern look. Various users prefer different visual modes depending on their environment and personal preferences. As a developer or designer, enabling dark mode based on user preferences can enhance the user experience, making your interface more adaptable and accessible.

This comprehensive guide will walk you through the techniques and best practices to enable dark mode dynamically by detecting user preferences and implementing it in both websites and applications.

Understanding Dark Mode and User Preferences

Before diving into the technical steps, it’s helpful to understand what dark mode entails and how user preferences are determined:

  • Dark Mode: A display setting that applies a darker color palette to user interfaces, reducing bright white backgrounds in favor of blacks, dark grays, or muted tones.
  • User Preferences: Many operating systems and browsers allow users to select whether they prefer a light or dark theme. This preference is often system-wide and can be leveraged by websites to enhance compatibility and user satisfaction.

Modern browsers pass user preference information through CSS media queries, primarily with the prefers-color-scheme media feature, allowing developers to style accordingly without explicit user input.

Step 1: Detecting User Dark Mode Preference with CSS

The simplest method to implement dark mode based on user preferences is using the CSS media query prefers-color-scheme. This detects the system-level theme the user has chosen.

/* Default light theme */
body {
  background-color: #ffffff;
  color: #000000;
}

/* Dark theme for users with dark mode preference */
@media (prefers-color-scheme: dark) {
  body {
    background-color: #121212;
    color: #e0e0e0;
  }
}

How It Works:

  • The default styles apply normally.
  • When the user prefers a dark color scheme (set at OS or browser level), the styles inside the media query override the defaults.

Pros:

  • Simple and requires no JavaScript.
  • Respects user system preferences automatically.
  • Supported by all modern browsers.

Cons:

  • No ability for the user to manually toggle themes within your app or website.
  • Scoped to CSS only; dynamic changes in theme require page reload to honor updated preferences.

Step 2: Allowing User-Controlled Theme Switching

While automatically detecting preferences is convenient, many users want control to toggle between light and dark modes regardless of their system settings. To do this, you’ll need to use JavaScript and save the user’s choice.

Implementing a Toggle Button

Create a button in your HTML that users can click to switch themes:

<button id="theme-toggle">Toggle Dark Mode</button>

JavaScript to Toggle Themes

const toggleButton = document.getElementById('theme-toggle');
const body = document.body;

// Check local storage for saved user theme
const savedTheme = localStorage.getItem('theme');

if (savedTheme) {
  body.classList.toggle('dark-mode', savedTheme === 'dark');
} else {
  // If no saved theme, apply based on system preference
  if(window.matchMedia && window.matchMedia('(prefers-color-scheme: dark)').matches) {
    body.classList.add('dark-mode');
  }
}

toggleButton.addEventListener('click', () => {
  const isDark = body.classList.toggle('dark-mode');
  localStorage.setItem('theme', isDark ? 'dark' : 'light');
});

CSS for Dark Mode Class

body.dark-mode {
  background-color: #121212;
  color: #e0e0e0;
}

What This Does:

  • Checks local storage for a user-selected theme and applies it.
  • Defaults to system setting if no prior user choice is saved.
  • Allows manual toggling with a button.
  • Saves user preference for future visits.

Step 3: Handling Dynamic Preference Changes

Some users change their system preference while browsing. To keep the theme in sync, listen for changes in prefers-color-scheme and update your theme dynamically, but only if the user hasn’t explicitly chosen a preference in your app.

const mediaQuery = window.matchMedia('(prefers-color-scheme: dark)');

function applySystemTheme() {
  if (!localStorage.getItem('theme')) {
    if (mediaQuery.matches) {
      body.classList.add('dark-mode');
    } else {
      body.classList.remove('dark-mode');
    }
  }
}

mediaQuery.addEventListener('change', applySystemTheme);
applySystemTheme();

Step 4: Incorporating Dark Mode in React or Other Frameworks

If your project uses React, Vue, Angular, or another front-end framework, managing dark mode often involves state management. The principles remain the same: detect preference, allow user override, persist settings, and respond to system changes.

Example: React Hook for Dark Mode

import { useState, useEffect } from 'react';

function useDarkMode() {
  const [theme, setTheme] = useState(() => {
    const saved = localStorage.getItem('theme');
    if(saved) return saved;
    return window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
  });

  useEffect(() => {
    document.body.classList.toggle('dark-mode', theme === 'dark');
    localStorage.setItem('theme', theme);
  }, [theme]);

  useEffect(() => {
    const mediaQuery = window.matchMedia('(prefers-color-scheme: dark)');
    const handleChange = () => {
      if(!localStorage.getItem('theme')) {
        setTheme(mediaQuery.matches ? 'dark' : 'light');
      }
    };
    mediaQuery.addEventListener('change', handleChange);
    return () => mediaQuery.removeEventListener('change', handleChange);
  }, []);

  return [theme, setTheme];
}

export default useDarkMode;

Usage:

function App() {
  const [theme, setTheme] = useDarkMode();

  return (
    <div>
      <button onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}>
        Switch to {theme === 'dark' ? 'Light' : 'Dark'} Mode
      </button>
      {/* Rest of your app */}
    </div>
  );
}

Step 5: Styling Considerations for Dark Mode

Designing dark mode isn’t only about inverting colors; it requires thoughtful adjustments to create a visually comfortable and accessible interface.

  • Contrast: Ensure text has adequate contrast against backgrounds.
  • Shadows and Highlights: Use subtle shadows to maintain depth without harsh brightness.
  • Images and Icons: Provide dark mode variants if your images don’t look good on dark backgrounds.
  • Link Colors: Use colors that stand out and have good visibility without being glaring.
  • Avoid pure black: Use very dark gray (#121212) as background for less eye strain.

Step 6: Improving Accessibility

Dark mode can improve accessibility for many users but may pose challenges for others.

  • Test with real users: Not every dark palette suits all users; gather feedback.
  • Provide easy toggles: Users should be able to override system defaults effortlessly.
  • Ensure keyboard usability: Make sure toggles and relevant UI elements are accessible by keyboard.
  • Screen readers: Confirm that adding dark-mode class or styles does not interfere with reading order or ARIA attributes.

Step 7: Advanced User Preference Handling with Server-Side Rendering

For server-rendered applications, you can enhance the user experience by detecting preferences on the server and rendering the appropriate theme initially. Since CSS media queries are client-side, you might need to infer preferences another way, such as:

  • Reading cookies storing user preference.
  • Using client hints if supported.
  • Falling back to default theme.

Setting the theme server-side reduces flicker and layout shifts when the page loads.

Conclusion

Enabling dark mode based on user preferences is a powerful way to make interfaces friendlier and more adaptive. By utilizing CSS media queries, offering manual toggles, listening for system preference changes, and considering accessibility and design patterns, you can create a seamless dark mode experience for your users.

Remember that dark mode is a user-centered feature; combining automatic detection with user control offers the most flexibility and satisfaction. Whether you work on websites or sophisticated front-end applications, the techniques outlined here provide the foundation to implement dark mode effectively.

With these strategies, you’ll not only improve usability but also keep your project at the forefront of modern UI trends. Dark mode isn’t just a fad—it’s a practical enhancement that users expect and appreciate. Get started today and create a comfortable viewing environment tailored to every visitor’s preferences!
