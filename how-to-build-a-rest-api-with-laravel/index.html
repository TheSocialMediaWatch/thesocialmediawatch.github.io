How to Build a REST API with Laravel


Laravel is one of the most popular PHP frameworks for developing modern web applications. With its clean and elegant syntax, Laravel makes building scalable, maintainable, and robust applications easier. One of the most common tasks in web development today is building a RESTful API, and Laravel provides an excellent set of tools to get you started quickly.

In this tutorial, we’ll walk you through the process of building a REST API with Laravel. Whether you are building a mobile app backend, a web application, or just want to expose some data to external services, this guide will take you step-by-step through everything you need to know.

Step 1: Setting Up Your Environment

Before we dive into the code, it’s important to set up the development environment properly. This includes ensuring that you have PHP, Composer, and Laravel installed on your system.

Installing Laravel

First, ensure that Composer is installed. Composer is a dependency management tool for PHP. You can check if Composer is installed by running the following command:

composer --version

If it’s not installed, you can install Composer by visiting Composer’s official website and following the installation instructions for your platform.

Once Composer is installed, you can create a new Laravel project. Open your terminal and run the following command:

composer create-project --prefer-dist laravel/laravel laravel-api

This will install a fresh Laravel project named laravel-api.

Configuring Environment

Next, navigate into the newly created project directory:

cd laravel-api

Open the .env file in the root directory and configure your database settings. For this example, let’s use MySQL. Update the database settings as follows:

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=your_database_name
DB_USERNAME=your_database_username
DB_PASSWORD=your_database_password

Make sure to replace the your_database_name, your_database_username, and your_database_password placeholders with your actual MySQL credentials.

Step 2: Setting Up Routes

Laravel uses a routing system to handle HTTP requests. For a REST API, we typically use HTTP verbs (GET, POST, PUT, DELETE) to perform CRUD (Create, Read, Update, Delete) operations.

Open the routes/api.php file. This file is where we’ll define the API routes. By default, Laravel includes a route for authentication, but we’ll define custom routes for our API endpoints.

Here’s an example of a basic set of routes for a simple Post resource:

use Illuminate\Http\Request;
use Illuminate\Support\Facades\Route;
use App\Http\Controllers\Api\PostController;

Route::get('/posts', [PostController::class, 'index']);
Route::get('/posts/{id}', [PostController::class, 'show']);
Route::post('/posts', [PostController::class, 'store']);
Route::put('/posts/{id}', [PostController::class, 'update']);
Route::delete('/posts/{id}', [PostController::class, 'destroy']);

These routes correspond to the typical CRUD operations:

  • GET /posts: Retrieves all posts.
  • GET /posts/{id}: Retrieves a single post by ID.
  • POST /posts: Creates a new post.
  • PUT /posts/{id}: Updates an existing post.
  • DELETE /posts/{id}: Deletes a post.

Step 3: Creating the Post Controller

Now that we’ve defined the routes, we need to create a controller to handle the logic for each route. Laravel provides a convenient way to generate controllers using Artisan, the built-in command-line tool.

To generate the PostController, run the following command:

php artisan make:controller Api/PostController

This will create a PostController in the app/Http/Controllers/Api directory. Now let’s open this file and add methods to handle the routes we’ve defined.

Here’s how the PostController might look:

namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use App\Models\Post;
use Illuminate\Http\Request;

class PostController extends Controller
{
    public function index()
    {
        $posts = Post::all();
        return response()->json($posts);
    }

    public function show($id)
    {
        $post = Post::find($id);
        if ($post) {
            return response()->json($post);
        } else {
            return response()->json(['message' => 'Post not found'], 404);
        }
    }

    public function store(Request $request)
    {
        $validated = $request->validate([
            'title' => 'required|string|max:255',
            'content' => 'required|string',
        ]);

        $post = Post::create($validated);
        return response()->json($post, 201);
    }

    public function update(Request $request, $id)
    {
        $post = Post::find($id);
        if (!$post) {
            return response()->json(['message' => 'Post not found'], 404);
        }

        $validated = $request->validate([
            'title' => 'required|string|max:255',
            'content' => 'required|string',
        ]);

        $post->update($validated);
        return response()->json($post);
    }

    public function destroy($id)
    {
        $post = Post::find($id);
        if (!$post) {
            return response()->json(['message' => 'Post not found'], 404);
        }

        $post->delete();
        return response()->json(['message' => 'Post deleted successfully']);
    }
}

Explanation:

  1. index: Fetches all posts from the database and returns them as a JSON response.
  2. show: Fetches a single post by ID. If the post exists, it returns the post as JSON; otherwise, it returns a 404 error.
  3. store: Accepts a POST request with a title and content for the post. It validates the data, creates a new post, and returns it as JSON.
  4. update: Accepts a PUT request to update a post by ID. If the post doesn’t exist, it returns a 404 error.
  5. destroy: Accepts a DELETE request to delete a post by ID. It returns a success message or a 404 error if the post does not exist.

Step 4: Creating the Post Model

In the example above, we’re using the Post model to interact with the database. To create this model, run the following Artisan command:

php artisan make:model Post -m

This command creates both the Post model and a migration file. The migration file is responsible for creating the database table. You can find the migration file in the database/migrations directory.

Open the migration file and define the schema for the posts table:

public function up()
{
    Schema::create('posts', function (Blueprint $table) {
        $table->id();
        $table->string('title');
        $table->text('content');
        $table->timestamps();
    });
}

Run the migration to create the table in your database:

php artisan migrate

Now, you have a posts table ready to store the post data.

Step 5: Testing the API

Laravel provides an elegant way to test APIs using tools like Postman or Insomnia. You can use these tools to send requests to your API and verify that everything works as expected.

Here’s a summary of the endpoints you can test:

  1. GET /api/posts: Retrieves all posts.
  2. GET /api/posts/{id}: Retrieves a post by ID.
  3. POST /api/posts: Creates a new post.
  4. PUT /api/posts/{id}: Updates an existing post.
  5. DELETE /api/posts/{id}: Deletes a post.

For each request, check the response code and the returned data. You should see JSON responses with appropriate status codes, like 200 OK for successful requests and 404 Not Found for missing resources.

Step 6: Adding Authentication (Optional)

For most real-world applications, you’ll want to add authentication to secure your API. Laravel provides multiple options for authentication, including API token-based authentication using Laravel Sanctum or Passport.

Using Laravel Sanctum for Simple API Authentication

  1. Install Sanctum:
composer require laravel/sanctum
  1. Publish the Sanctum configuration file:
php artisan vendor:publish --provider="Laravel\Sanctum\SanctumServiceProvider"
  1. Run the Sanctum migration:
php artisan migrate
  1. In app/Http/Kernel.php, add Sanctum’s middleware to the api middleware group:
'api' => [
    \Laravel\Sanctum\Http\Middleware\EnsureFrontendRequestsAreStateful::class,
    'throttle:api',
    \Illuminate\Routing\Middleware\SubstituteBindings::class,
],
  1. Protect the routes in your PostController by applying Sanctum’s auth:sanctum middleware:
Route::middleware('auth:sanctum')->get('/posts', [PostController::class, 'index']);

Now, only authenticated users will be able to access these API routes.

Conclusion

Building a REST API with Laravel is simple and straightforward, thanks to its elegant routing system, powerful controllers, and rich set of built-in features. In this tutorial, we’ve covered the essentials of setting up a Laravel project, defining routes, creating controllers, and interacting with a database through models.

With these foundations in place, you can easily extend your API by adding authentication, validation, rate limiting, and more. Laravel’s ecosystem has everything you need to build a powerful and secure API for your applications.
