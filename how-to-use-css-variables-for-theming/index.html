How to Use CSS Variables for Theming


In modern web development, maintaining consistent styles across a website or application is a significant challenge, especially as projects grow in scale and complexity. One of the most powerful tools in the CSS arsenal to tackle this problem is CSS variables, also known as custom properties. These variables allow developers to define reusable values that can be dynamically changed, making them perfect for implementing flexible, maintainable theming systems.

This article explores the concept of CSS variables, how they work, and practical approaches to use them for creating versatile and dynamic themes. Whether you’re building a dark mode toggle, branding multiple client sites, or simply want to organize your styles better, this guide will walk you through everything you need to know to get started.

What Are CSS Variables?

CSS variables are custom properties defined within CSS that store values to be reused throughout the stylesheet. Unlike preprocessor variables (such as those in Sass or LESS), CSS variables are native to the browser and can be manipulated dynamically using JavaScript or CSS itself.

Defining CSS Variables

CSS variables are declared within a selector using the syntax:

--variable-name: value;

Typically, variables are defined under the :root pseudo-class, which represents the highest-level element in the document (usually the <html> element). This scope allows variables to be globally accessible.

:root {
  --primary-color: #3498db;
  --secondary-color: #2ecc71;
  --font-size-base: 16px;
}

Using CSS Variables

To use a variable, the var() function references the custom property:

button {
  background-color: var(--primary-color);
  font-size: var(--font-size-base);
}

Using variables makes updating color schemes, font sizes, spacings, and other design tokens effortless.

Why Use CSS Variables for Theming?

Theming websites often involves swapping colors, fonts, and other styles to maintain brand identities or support user preferences like dark mode. CSS variables shine in this scenario for several reasons:

1. Dynamic Updates

Because CSS variables exist in the browser’s runtime, you can change their values dynamically via JavaScript or by changing a class on a parent element. This makes switching themes instant and efficient.

2. Ease of Maintenance

When you’ve centralized defining colors, sizes, and other design tokens as variables, updating your theme becomes straightforward. Instead of hunting down every instance of a color value, you modify one variable.

3. Inheritance and Scoping

CSS variables inherit through the DOM tree, enabling scoped theming. This makes it possible to have multiple themes coexisting on a single page or nested components each with their own theme.

Setting Up a Basic Theming System Using CSS Variables

Let’s walk through creating a simple light and dark theme toggle using CSS variables.

Step 1: Define Variables for Both Themes

Start by defining light theme variables globally with :root, which will serve as the default.

:root {
  --bg-color: #ffffff;
  --text-color: #222222;
  --link-color: #1a73e8;
  --button-bg: #1a73e8;
  --button-text: #ffffff;
}

Next, create a dark theme by overriding the variables under a .dark-theme selector:

.dark-theme {
  --bg-color: #121212;
  --text-color: #e0e0e0;
  --link-color: #8ab4f8;
  --button-bg: #8ab4f8;
  --button-text: #121212;
}

Step 2: Apply Variables in Styles

Apply these variables to your UI components:

body {
  background-color: var(--bg-color);
  color: var(--text-color);
  font-family: Arial, sans-serif;
  transition: background-color 0.3s ease, color 0.3s ease;
}

a {
  color: var(--link-color);
}

button {
  background-color: var(--button-bg);
  color: var(--button-text);
  border: none;
  padding: 0.6em 1.2em;
  cursor: pointer;
  border-radius: 4px;
  transition: background-color 0.3s ease, color 0.3s ease;
}

Step 3: Toggle the Theme Using JavaScript

Use JavaScript to switch between themes by toggling the .dark-theme class on the <body> or <html> tag:

const toggleBtn = document.getElementById('theme-toggle');

toggleBtn.addEventListener('click', () => {
  document.body.classList.toggle('dark-theme');
});

With this method, switching themes swaps out the variable values instantly and smoothly, thanks to CSS transitions.

Advanced Theming Techniques with CSS Variables

Once you become comfortable with basic theming, you can explore more powerful techniques that enhance flexibility and performance.

Scoped Themes for Components

CSS variables inherit through the DOM, which means you can scope themes to individual components or sections in a page. This is great for embedded widgets, previews, or multi-brand portals.

.widget-light {
  --bg-color: #f7f7f7;
  --text-color: #333333;
}

.widget-dark {
  --bg-color: #2c2c2c;
  --text-color: #cccccc;
}

.widget {
  background-color: var(--bg-color);
  color: var(--text-color);
  padding: 1em;
  border-radius: 6px;
}

HTML snippet:

<div class="widget widget-light">Light Themed Widget</div>
<div class="widget widget-dark">Dark Themed Widget</div>

Responsive Theming Based on Media Queries

You can use CSS media queries to adapt themes based on user preferences, such as prefers-color-scheme.

@media (prefers-color-scheme: dark) {
  :root {
    --bg-color: #121212;
    --text-color: #E0E0E0;
    --link-color: #bb86fc;
  }
}

This enhances user experience by automatically applying themes aligned with their system preferences.

Using Variables for More Than Colors

CSS variables are not just for colors. They can store typography sizes, spacings, shadows, animation timings, and more. A robust theming system may include all of these tokens.

:root {
  --font-size-base: 16px;
  --spacing-unit: 8px;
  --border-radius: 12px;
  --transition-speed: 0.3s;
}

.card {
  padding: calc(var(--spacing-unit) * 2);
  border-radius: var(--border-radius);
  font-size: var(--font-size-base);
  transition: box-shadow var(--transition-speed);
}

When you create multiple themes, this can control the entire UI’s feel, not just colors.

Managing CSS Variables at Scale

In larger projects, managing hundreds of variables can become unwieldy. Consider these strategies for maintainable, scalable theming architecture.

Organize Variables in Logical Groups

Group variables by their purpose or UI domain:

:root {
  /* Colors */
  --color-primary: #3498db;
  --color-secondary: #2ecc71;
  
  /* Typography */
  --font-family-base: "Roboto", sans-serif;
  --font-size-base: 18px;
  
  /* Spacing */
  --spacing-small: 8px;
  --spacing-medium: 16px;
  
  /* Shadows */
  --shadow-light: 0 1px 3px rgba(0,0,0,0.12);
}

Use Preprocessing or Build Tools for Variable Generation

While CSS variables are native to CSS, you can combine them with preprocessors or build tools to automate variable generation or import variables from design tokens.

For example, using tools like Style Dictionary to maintain centralized design tokens that generate CSS variables compatible with your codebase.

Document Variables Clearly

Maintain a living style guide or documentation site listing all CSS variables, their purposes, acceptable values, and theme variants to aid collaboration.

Dynamically Changing Themes Beyond CSS Classes

Beyond toggling classes, you can manipulate CSS variables directly via JavaScript to create more nuanced theming controls or user-customizable themes.

Example: Changing Primary Color on User Input

<label for="colorPicker">Pick Primary Color:</label>
<input type="color" id="colorPicker" name="colorPicker" value="#3498db" />

const colorPicker = document.getElementById('colorPicker');

colorPicker.addEventListener('input', (event) => {
  const newColor = event.target.value;
  document.documentElement.style.setProperty('--primary-color', newColor);
});

This approach offers a highly flexible user experience where colors, font sizes, or other design properties adjust live.

Benefits

  • Allows granular, real-time customization without recalculations.
  • Can be combined with CSS variables to impact the entire design.
  • Empowers tools for accessible and user-friendly theming.

Troubleshooting Common Pitfalls with CSS Variables for Theming

Variables Not Applying as Expected

  • Make sure the variable is defined within the correct scope or ancestor.
  • Check for syntax errors; variable names are case sensitive and must be prefixed with --.
  • Ensure the fallback value in var() is provided if variable might be undefined: var(--some-var, fallback-value).

Transitioning Variables Smoothly

Animating CSS variables requires the property using the variable to be animatable. For example, animating colors works well, but you cannot animate font-family.

Browser Compatibility

Most modern browsers support CSS variables widely, but if your project needs to support legacy browsers (like IE11), you must provide fallbacks or avoid variables.

Conclusion

CSS variables open up new horizons for creating flexible, maintainable, and dynamic themes. From simple light/dark switches to complex multi-brand theming setups, CSS custom properties make it easier than ever to centralize design tokens, respond to user preferences, and optimize your styles for scalability.

By defining meaningful variables scoped globally or at the component level, leveraging media queries for preferred user settings, and integrating JavaScript for dynamic customizations, you can build elegant theming solutions that enhance user experience and development efficiency.

Experiment with variables for colors, typography, spacing, and beyond to truly harness their power in taking your UI design and theming capabilities to the next level.

Ready to transform your styling workflow? Start by identifying core design tokens in your current CSS and migrating them into CSS variables. From there, build up themes that empower users and simplify your code, one variable at a time.
