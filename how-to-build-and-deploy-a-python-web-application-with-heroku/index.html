How to Build and Deploy a Python Web Application with Heroku


Creating a Python web application and making it accessible online might seem daunting at first, but with modern tools like Heroku, the process is surprisingly smooth and efficient. This guide covers everything you need to build a Python web app from scratch and deploy it on Heroku, a cloud platform that simplifies app deployment, scaling, and management.

Setting the Stage: Why Use Heroku for Python Web Apps?

Heroku offers an intuitive platform-as-a-service (PaaS) environment that abstracts away much of the heavy lifting involved in deployment. Instead of managing infrastructure manually, developers focus on writing code, while Heroku handles servers, provisioning, networking, and scaling.

With seamless Git integration and built-in support for Python, deploying a web application becomes a straightforward process. Plus, Heroku supports popular web frameworks in Python like Flask and Django, which can be quickly set up and deployed.

Step 1: Preparing Your Development Environment

To get started, ensure you have the following installed on your local machine:

  • Python 3.x: The latest stable version is recommended.
  • pip: Python’s package installer.
  • Virtualenv or venv: For creating isolated Python environments.
  • Git: Essential for version control and Heroku deployment.
  • Heroku CLI: The command-line interface tool used to manage Heroku apps.

Installing Necessary Tools

  • Python: Download from python.org.
  • Git: Available at git-scm.com.
  • Heroku CLI: Installation instructions can be found here.

Once installed, verify by running:

python --version
git --version
heroku --version

Step 2: Creating a Python Web Application

For this tutorial, let’s build a simple web application using Flask, a lightweight and easy-to-learn Python web framework.

Initializing the Project

Create a new directory for your project and navigate into it:

mkdir my-flask-app
cd my-flask-app

Set up a virtual environment to keep dependencies isolated:

python -m venv venv
source venv/bin/activate  # On Windows, use: venv\Scripts\activate

Installing Flask

Inside the virtual environment, run:

pip install Flask gunicorn
  • Flask serves as the web framework.
  • Gunicorn is a Python HTTP server needed for running the app in production environments like Heroku.

Writing the Application Code

Create a file named app.py:

from flask import Flask

app = Flask(__name__)

@app.route('/')
def home():
    return "Welcome to my Flask app deployed on Heroku!"

if __name__ == '__main__':
    app.run(debug=True)

This minimal Flask app defines a root route that responds with a friendly message.

Step 3: Preparing the Application for Heroku

Heroku requires a few extra files to configure the deployment correctly.

1. requirements.txt

This file lists all Python dependencies to be installed on the Heroku server.

Generate it with:

pip freeze > requirements.txt

Make sure it includes Flask and gunicorn.

2. Procfile

This file tells Heroku how to start your application. Create it at the root of your project:

web: gunicorn app:app

This instructs Heroku to use Gunicorn to run the app object from app.py.

3. runtime.txt (optional)

Specify the Python version to ensure consistency between local and Heroku environments:

python-3.11.4

Replace the version with your preferred Python version.

Step 4: Version Control with Git

Initialize a Git repository, add all your files, and commit:

git init
git add .
git commit -m "Initial commit - Flask app setup for Heroku"

Step 5: Deploying to Heroku

Logging into Heroku

Run:

heroku login

A browser window will open for authentication.

Creating a Heroku App

Create a new Heroku app:

heroku create

Heroku will assign a unique URL and git remote (commonly named heroku).

Alternatively, specify a name:

heroku create my-flask-demo

Pushing Code to Heroku

Deploy the application by pushing your Git repository:

git push heroku main

Note: Use master if your main Git branch is named master.

During the first deployment, Heroku installs dependencies from requirements.txt, sets up the environment, and runs your app using the Procfile.

Step 6: Accessing Your Application

Once deployment completes, open your live app:

heroku open

You should see the message “Welcome to my Flask app deployed on Heroku!” displayed in your browser.

Step 7: Managing Environment Variables

For any production app, hardcoding sensitive information (API keys, database URIs) is discouraged. Heroku supports config vars that can be accessed like environment variables.

Set a config var with:

heroku config:set SECRET_KEY='your-secret-key'

Access the variable in Python with:

import os

secret_key = os.environ.get('SECRET_KEY')

This approach secures sensitive data and supports different values per environment.

Step 8: Adding a Database (Optional)

Most real-world web applications require persistent storage. Heroku provides various add-ons, such as Heroku Postgres.

Adding Postgres to Your App

Run:

heroku addons:create heroku-postgresql:hobby-dev

Retrieve the database URL supplied by Heroku:

heroku config:get DATABASE_URL

Use this URL in your application to connect to PostgreSQL.

Using SQLAlchemy with Flask

To interact with a PostgreSQL database, developers often use SQLAlchemy, a powerful Object-Relational Mapper (ORM):

pip install flask_sqlalchemy psycopg2-binary

Add the following to your app.py:

from flask_sqlalchemy import SQLAlchemy
import os

app.config['SQLALCHEMY_DATABASE_URI'] = os.environ.get('DATABASE_URL')
app.config['SQLALCHEMY_TRACK_MODIFICATIONS'] = False

db = SQLAlchemy(app)

class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(80), unique=True, nullable=False)

@app.before_first_request
def create_tables():
    db.create_all()

This setup initializes the database and creates tables upon the first request.

Step 9: Scaling Your Application

Heroku dynos are the containers running your app. By default, one web dyno runs your application.

Scale dynos:

heroku ps:scale web=2

This command doubles your app’s capacity to handle traffic.

Step 10: Monitoring and Logs

Understanding your app’s performance and errors is critical.

  • View logs live:
heroku logs --tail

This streams logs continuously.

  • More in-depth monitoring and error tracking can be enabled with additional Heroku add-ons or external services like Sentry.

Helpful Tips and Best Practices

  • Use .gitignore to exclude files such as venv/, .env, and __pycache__/.
  • Pin dependencies in requirements.txt using exact versions to avoid unexpected breakages.
  • Test locally with Gunicorn before pushing to Heroku:
gunicorn app:app
  • Automate deployment with GitHub Actions or other CI/CD tools that integrate with Heroku.

Conclusion

Deploying a Python web application on Heroku blends simplicity with powerful scalability. By using Flask and Gunicorn, preparing the necessary configuration files, and mastering Git and Heroku CLI commands, you can have a robust web app up and running in no time. From local development to full production deployment, Heroku manages the heavy infrastructure lifting so you can focus on what matters most: building great applications. With this foundational knowledge, expanding to databases, scaling traffic, and integrating additional features becomes a natural progression in your web development journey.
